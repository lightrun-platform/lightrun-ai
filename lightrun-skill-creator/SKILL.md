# Goal

Produce runtime-oriented Lightrun skills that are deterministic, concise, and auditable.

# Scope

- In scope: runtime-skill authoring rules, tool-name correctness, flow structure, dependency metadata shape, and quality checklist.
- Out of scope: executing runtime investigations or changing product code.

# Preconditions

- Author can edit skill files.
- Generated runtime skills include MCP preflight.
- Generated runtime skills include Lightrun MCP dependency metadata.

# MCP Preflight

- Required gate tool:
  - `lightrun__get_runtime_sources`
- Pass criteria:
  - Gate appears before runtime evidence tools.
  - Tool name is exact.
- Fail criteria:
  - Gate missing, misplaced, or renamed.

# Flow

1. Define skill intent and success outcome.
   - Tools: none
   - Success: clear goal and terminal output.
2. Add deterministic preflight and recovery branch.
   - Tools: `lightrun__get_runtime_sources`
   - Success: explicit pass/fail behavior and resume condition.
3. Select runtime tools per step objective.
   - Tools: minimal subset from canonical list.
   - Success: each step lists only purpose-driven tools.
4. Add generated-skill metadata rules.
   - Tools: none
   - Success: generated skills have valid `agents/openai.yaml` shape and OAuth note in generated `SKILL.md`.
5. Finalize output contract and checklist.
   - Tools: none
   - Success: preflight pass/fail and runtime blocker outputs are explicit.

# Missing-MCP Recovery

1. Classify failure: missing tool, tool call error, or empty sources.
2. In generated runtime skill docs, require Lightrun MCP installation and OAuth authorization.
3. Re-run preflight.
4. Continue after gate success.

# Resume Criteria

- Resume flow after `lightrun__get_runtime_sources` succeeds.
- Runtime evidence tools are enabled after gate success.

# Output Contract

- Preflight pass: selected source and next action.
- Preflight fail: blocker + required remediation.
- Runtime blocker: failed tool + reason + next action.
- Final handoff: file path + checklist status.

# Definition of Done Checklist

- [ ] Required sections exist and are coherent.
- [ ] Canonical `lightrun__*` names are exact.
- [ ] Generated runtime skills use minimal required tools only.
- [ ] Generated runtime skills include valid MCP dependency metadata.
- [ ] Generated runtime skills document OAuth requirement in `SKILL.md`.
- [ ] Generated runtime skills inherit icon metadata values from this creator skill.
- [ ] Output contract is specific and action-oriented.

## Canonical Runtime Tool List

- `lightrun__get_runtime_sources`
- `lightrun__get_runtime_expression_values`
- `lightrun__get_runtime_execution_duration`
- `lightrun__get_runtime_execution_duration_samples`
- `lightrun__get_runtime_numeric_metric`
- `lightrun__get_runtime_numeric_metric_samples`
- `lightrun__get_runtime_callstack`
- `lightrun__get_runtime_execution_count`

## Generated Skill Metadata Requirements

For each runtime skill generated by this creator, require `agents/openai.yaml` with these top-level blocks:

- `interface`
- `dependencies`

Required interface fields in generated skill:

- `interface.icon_small` (inherit from this creator skill value)
- `interface.icon_large` (inherit from this creator skill value)
- `interface.brand_color` (inherit from this creator skill value)

Required MCP dependency entry in generated skill:

```yaml
dependencies:
  tools:
    - type: "mcp"
      value: "lightrun"
      description: "Lightrun MCP server"
      transport: "streamable_http"
      url: "https://app.lightrun.com/mcp"
```
